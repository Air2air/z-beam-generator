# Table Component - YAML Output Example

## Next.js Usage
```javascript
import yaml from 'js-yaml';

// Load the generated YAML file
const tableData = yaml.load(yamlContent);

// Render tables
{tableData.materialTables.tables.map((table, index) => (
  <div key={index} className="mb-8">
    <h2 className="text-xl font-bold mb-4">
      {table.header.replace('## ', '')}
    </h2>
    <table className="min-w-full border border-gray-300">
      <thead className="bg-gray-50">
        <tr>
          <th className="px-4 py-2 text-left">Property</th>
          <th className="px-4 py-2 text-left">Value</th>
          <th className="px-4 py-2 text-left">Range</th>
          <th className="px-4 py-2 text-left">Percentile</th>
          <th className="px-4 py-2 text-left">Visualization</th>
        </tr>
      </thead>
      <tbody>
        {table.rows.map((row, rowIndex) => (
          <tr key={rowIndex} className="border-b">
            <td className="px-4 py-2 font-medium">{row.property}</td>
            <td className="px-4 py-2">{row.value} {row.unit}</td>
            <td className="px-4 py-2">
              {row.min && row.max ? `${row.min}-${row.max}` : '-'}
            </td>
            <td className="px-4 py-2">
              {row.percentile ? `${row.percentile}%` : 'N/A'}
            </td>
            <td className="px-4 py-2">
              <div dangerouslySetInnerHTML={{ __html: row.htmlVisualization }} />
            </td>
          </tr>
        ))}
      </tbody>
    </table>
  </div>
))}
```

## Key Features

1. **Categorized Properties**: Automatically groups properties into logical categories
2. **Interactive Visualizations**: HTML snippets with Tailwind CSS for progress bars
3. **Range Support**: Handles min/max values with percentage calculations
4. **Next.js Optimized**: YAML structure designed for easy React/Next.js consumption
5. **Responsive Design**: Tailwind CSS classes for mobile-friendly tables
6. **Deterministic Output**: Same frontmatter input always produces identical YAML
